# Analyze Backlog - Options Reference

## Script Commands

The `analyze_backlog.py` script provides three commands:

### find - Find Unanalyzed Issues

Find backlog issues that don't have the `claude-analyzed` label.

```bash
python scripts/analyze_backlog.py find PROJECT [options]
```

| Option | Description |
|--------|-------------|
| `PROJECT` | Required. Jira project key (e.g., EFT) |
| `--max-results`, `-n` | Max issues to return (default: 3) |
| `--format`, `-f` | Output format: compact, text (default), json |

**Examples:**

```bash
# Find top 3 unanalyzed issues
python scripts/analyze_backlog.py find EFT

# Find top 5, JSON output
python scripts/analyze_backlog.py find EFT -n 5 -f json
```

### get - Get Full Issue Details

Retrieve complete issue details including description for analysis.

```bash
python scripts/analyze_backlog.py get ISSUE-KEY [options]
```

| Option | Description |
|--------|-------------|
| `ISSUE-KEY` | Required. Jira issue key (e.g., EFT-123) |
| `--format`, `-f` | Output format: compact, text (default), json |

**Examples:**

```bash
# Get issue details
python scripts/analyze_backlog.py get EFT-123

# Get as JSON for parsing
python scripts/analyze_backlog.py get EFT-123 -f json
```

### update - Update Issue with Analysis

Update an issue by appending analysis to the description and adding the `claude-analyzed` label.

```bash
python scripts/analyze_backlog.py update ISSUE-KEY [options]
```

| Option | Description |
|--------|-------------|
| `ISSUE-KEY` | Required. Jira issue key (e.g., EFT-123) |
| `--analysis` | Analysis text to append (inline) |
| `--analysis-file` | Path to file containing analysis text |
| `--format`, `-f` | Output format: compact, text (default), json |

**Examples:**

```bash
# Update with inline analysis
python scripts/analyze_backlog.py update EFT-123 --analysis "### Summary\nImplement feature X..."

# Update from file
python scripts/analyze_backlog.py update EFT-123 --analysis-file /tmp/analysis.md
```

## Output Formats

### compact

Minimal token usage, pipe-delimited:

```
EFT-123|Open|vue,frontend|Implement dark mode toggle
```

### text (default)

Human-readable multi-line format:

```
Issue: EFT-123
Summary: Implement dark mode toggle
Status: Open
Type: Task
...
```

### json

Compact JSON for programmatic use:

```json
{"key":"EFT-123","summary":"Implement dark mode toggle","status":"Open"}
```

## Label Convention

The skill uses the `claude-analyzed` label to track which issues have been analyzed:

- **Before analysis**: Issue does NOT have `claude-analyzed` label
- **After analysis**: Issue HAS `claude-analyzed` label

**Note:** The label is automatically created in Jira when first applied - no manual setup required. Jira creates labels on-the-fly when added to any issue.

## Description Update Format

When updating an issue, the analysis is appended to preserve the original description:

```
[Original Description]

---

## Claude Analysis

[Analysis Content]

---
*Analysis generated by Claude on YYYY-MM-DD*
```

The horizontal rules (`---`) create visual separation in Jira's display.

## Analysis Content Structure

The analysis section should follow this structure:

```markdown
### Summary
One-sentence description of what will be implemented.

### Pre-conditions
- Dependency 1
- Setup requirement 2

### Implementation Steps

#### 1. First Step
**Files:** `path/to/file.ext`

Description of the step.

```pseudo
// Pseudo-code
function example():
  logic here
```

#### 2. Second Step
...

### Testing Strategy
- Test approach 1
- Test approach 2

### Estimated Scope
- Files to modify: X
- New files: Y
- Complexity: Low/Medium/High
```

## Environment Variables

Required for all operations:

| Variable | Description |
|----------|-------------|
| `JIRA_BASE_URL` | Jira instance URL (e.g., `https://yoursite.atlassian.net`) |
| `JIRA_EMAIL` | Your Jira account email |
| `JIRA_API_TOKEN` | API token from Atlassian account settings |

## Error Handling

Common errors and solutions:

| Error | Cause | Solution |
|-------|-------|----------|
| `Authentication failed` | Invalid credentials | Check JIRA_EMAIL and JIRA_API_TOKEN |
| `Issue not found` | Invalid issue key | Verify the issue key exists |
| `Access denied` | Permission issue | Ensure you have edit access to the issue |
| `No unanalyzed issues` | All issues have label | Analysis complete or check label |

## Cache Integration

The script integrates with the shared jira-tools cache:

- Updates cached issue data after modifications
- Uses cache for user and label lookups
- Cache location: `~/.jira-tools-cache.json`

Manage cache:

```bash
python shared/jira_cache.py info    # View cache status
python shared/jira_cache.py clear   # Clear all cache
```
